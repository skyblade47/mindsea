# AI 肉鸽文字冒险 — 统一创作架构设计（v3.0）

> 核心转变：**不再区分"技能创作"和"装备锻造"，只有一个创作系统。**
>
> 资源决定强度上限、提示词决定具体表现、时间决定代价。
> AI不是工具，是游戏循环中的一个"铸造环节"。

---

## 一、核心方程

```
创作结果 = f(资源品质, 提示词精度, 投入时间)

资源品质 → 决定强度的天花板（你投入了多好的材料）
提示词精度 → 决定效果的具体表现（你描述得多清楚）
投入时间 → 决定这次铸造消耗的冒险时间
```

---

## 二、资源系统：碎片素材

### 2.1 碎片来源

| 来源 | 掉落类型 | 品质范围 |
|------|---------|---------|
| 普通战斗 | 元素碎片（火/冰/雷/暗影...） | 普通-稀有 |
| 精英战斗 | 机制碎片（穿透/吸血/反弹/连锁...） | 稀有-史诗 |
| BOSS 战 | 核心碎片（因果/时空/生命/存在...） | 史诗-传说 |
| 探索事件 | 古代提示词碎片（预设技能） | 固定 |
| 商人 | 任意碎片（可购买） | 随机 |

### 2.2 碎片类型

| 大类 | 细分 | 作用 |
|------|------|------|
| **元素碎片** | 火、冰、雷、暗影、神圣、自然、虚空 | 决定技能的属性倾向 |
| **形态碎片** | 单体、范围、穿透、连锁、持续、爆发 | 决定技能的作用形式 |
| **机制碎片** | 吸血、反弹、护盾、净化、召唤、复制 | 决定技能的特殊效果 |
| **品质碎片** | 凡铁、精钢、秘银、源质、神骸 | 提升整体强度上限 |

### 2.3 碎片组合决定强度

```
单个技能可投入的碎片总量 = 品质碎片的承载上限

凡铁碎片：承载上限 = 3
精钢碎片：承载上限 = 5
秘银碎片：承载上限 = 8
源质碎片：承载上限 = 12
神骸碎片：承载上限 = 20
```

### 2.4 碎片是消耗品

- 每创造一个技能，投入的碎片被消耗
- 冒险结束后，未使用的碎片**清零**（肉鸽机制）
- 局外解锁可以增加"开局携带碎片"的数量

## 三、提示词系统：表达决定表现

### 3.1 提示词的三种形态

| 形态 | 输入方式 | 效果 | 场景 |
|------|---------|------|------|
| **纯碎片** | 只放碎片，不写描述 | 基础效果，AI自行决定表现形式 | 快速生成，省时间 |
| **碎片 + 描述** | 放碎片 + 补充自然语言 | 碎片决定强度，描述决定精准度 | **标准玩法** |
| **纯描述** | 不依赖碎片，纯自然语言 | AI完全自由发挥（强度浮动大） | 创意探索 |

### 3.2 自然语言加成机制

```
加成系数 = 1 + (自然语言占比 × 效果倍率)

自然语言占比 = 描述字数 / (描述字数 + 碎片等效字数)
  - 每个碎片等效为 10 字（预设内容）
```

## 四、统一后的顶层架构

### 4.1 新架构模块图

```
统一创作引擎 (Unified Creation Engine)
├── 碎片管理系统 (Fragment Manager)
│   ├── 掉落表
│   ├── 库存（单局）
│   ├── 组合规则
│   └── 承载上限计算
│
├── 提示词系统 (Prompt System)
│   ├── 碎片+描述输入界面
│   ├── 自然语言加成计算
│   ├── 纯碎片/混合/纯描述 三种模式
│   └── 生成预览
│
├── 古代提示词 (Ancient Lexicon)
│   ├── 剧情采集
│   ├── 记事本（单局）
│   ├── 完整性检查
│   └── 收藏本（跨局）
│
└── AI铸造 (AI Smith)
    ├── Prompt组装（碎片信息 + 玩家描述 + 约束）
    ├── 3变体生成
    ├── 规则验证（L0骨架）
    └── 结果输出
```
