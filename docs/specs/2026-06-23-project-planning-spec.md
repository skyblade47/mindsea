# AI 文字冒险游戏 — 项目规划文档

## 一、项目概述

### 核心概念
一款**文字冒险游戏**，核心卖点是**玩家用 AI 创造专属技能**。玩家通过冒险获得经验、升级、赚取技能点，再消耗技能点让 AI 根据自然语言描述生成个性化技能。技能越强、越通用，消耗的技能点越多。

### 一句话描述
> "用文字编织你的力量——在 AI 驱动的冒险世界里，你的想象力就是最强的技能。"

### 目标平台
Web 端（PC + 移动端自适应）

### 开发周期（MVP）
4–6 周

## 二、核心游戏机制

### 2.1 游戏循环

```
开始冒险 → 遭遇事件/敌人 → 做选择/使用技能 → 获得经验/奖励 →
升级 → 获得技能点 → 创作/强化技能 → 继续冒险
```

### 2.2 玩家基础属性（MVP 简化版）

| 属性 | 作用 |
|------|------|
| **生命值 (HP)** | 归零则冒险失败，回到存档点 |
| **精神力 (MP)** | 使用技能消耗，休息或道具恢复 |
| **等级 (Lv)** | 决定基础能力上限，影响技能点获取 |

### 2.3 经验与升级

| 等级 | 升级所需经验 | 获得技能点 |
|------|-------------|-----------|
| 1→2 | 100 | +3 |
| 2→3 | 250 | +3 |
| 3→4 | 500 | +4 |
| 4→5 | 900 | +4 |
| 5+ | 前一级 × 1.6 | +5 |

### 2.4 冒险事件类型

1. **战斗事件** — 遭遇敌人，使用技能战斗
2. **探索事件** — 发现宝藏、秘密通道、NPC
3. **抉择事件** — 道德/策略选择，影响后续剧情
4. **休息事件** — 恢复 HP/MP，整理技能
5. **Boss 战** — 章节终点，击败后进入新区域

## 三、技能系统（核心卖点）

### 3.1 技能创作流程

```
玩家输入技能名称 + 描述 → AI 解析效果 → AI 评估强度 →
显示所需技能点 → 玩家确认消耗 → 技能生成并加入技能库
```

### 3.2 技能信息结构

```json
{
  "id": "skill_001",
  "name": "火焰风暴",
  "description": "召唤一场烈焰风暴，焚烧周围所有敌人，并点燃地面持续造成伤害",
  "category": "攻击",
  "target": "全体敌人",
  "effect": {
    "immediate": "造成 150% 智力值的火焰伤害",
    "dot": "每回合额外 30% 火焰伤害，持续 3 回合",
    "area": "点燃地面，敌人经过时受到 20% 伤害"
  },
  "cost": {
    "mp": 40,
    "skillPoints": 12
  },
  "cooldown": 3,
  "rarity": "稀有",
  "creator": "玩家ID"
}
```

### 3.3 技能强度评估维度

AI 从以下 6 个维度评估技能，每个维度 1–10 分：

| 维度 | 说明 | 高分代表 |
|------|------|---------|
| **威力 (Power)** | 伤害/治疗/护盾的数值强度 | 高伤害、强力治疗 |
| **范围 (Scope)** | 影响的目标数量 | 全体攻击、团队增益 |
| **持续性 (Duration)** | 效果持续的时间 | 长期 buff、持续伤害 |
| **灵活性 (Flexibility)** | 适用场景的广泛程度 | 战斗/探索/解谜通用 |
| **独特性 (Uniqueness)** | 是否拥有特殊机制 | 控制、传送、复活等 |
| ** combo 潜力 (Synergy)** | 与其他技能配合的强度 | 触发连锁反应 |

### 3.4 技能点消耗计算公式

```
基础消耗 = 3
维度总分 = 威力 + 范围 + 持续性 + 灵活性 + 独特性 + combo潜力
消耗系数 = 维度总分 / 6  (平均值)

最终消耗 = 基础消耗 + (消耗系数 × 2) + 独特性加成

其中：
- 独特性 ≥ 8 时，额外 +3 点
- 灵活性 = 10 时，额外 +2 点（万精油惩罚）
- 范围 = 10 且 威力 ≥ 7 时，额外 +4 点（AOE高伤惩罚）

最终消耗范围：4–25 技能点
```

**示例评估：**

| 技能描述 | 威力 | 范围 | 持续 | 灵活 | 独特 | combo | 总分 | 消耗 |
|---------|------|------|------|------|------|-------|------|------|
| "挥剑攻击一个敌人" | 5 | 3 | 1 | 3 | 2 | 2 | 16 | 5点 |
| "治疗自己少量生命" | 4 | 2 | 1 | 4 | 3 | 2 | 16 | 5点 |
| "召唤陨石轰击全体敌人，并眩晕 1 回合" | 9 | 10 | 3 | 5 | 9 | 7 | 43 | 20点 |
| "让自己可以在墙壁间瞬移" | 2 | 2 | 5 | 10 | 9 | 6 | 34 | 16点 |

### 3.5 技能使用限制

- **MP 消耗**：使用技能消耗精神力
- **冷却回合**：强力技能有使用冷却
- **场景限制**：某些技能只能在特定场景使用（如"水下呼吸"只能在水中）
- **稀有度上限**：低等级玩家无法创作高消耗技能（防止一开始就用 OP 技能）

### 3.6 技能强化

已拥有的技能可以消耗额外技能点进行强化：
- **威力强化**：伤害/效果 +20%，消耗原技能 50% 点数
- **范围扩展**：单体→群体，消耗原技能 100% 点数
- **冷却缩减**：冷却 -1，消耗原技能 75% 点数

## 四、AI 系统架构

### 4.1 AI 职责分工

| 模块 | AI 职责 | 调用时机 |
|------|---------|---------|
| **剧情生成** | 根据当前进度生成下一段冒险剧情 | 玩家推进时 |
| **事件生成** | 随机生成战斗/探索/抉择事件 | 进入新场景时 |
| **敌人生成** | 根据玩家等级生成合适难度的敌人 | 战斗事件触发时 |
| **技能解析** | 将玩家自然语言转化为结构化技能数据 | 技能创作时 |
| **强度评估** | 对技能进行多维度评分 | 技能解析后 |
| **战斗模拟** | 根据技能效果计算战斗结果 | 玩家使用技能时 |
| **NPC 对话** | 生成 NPC 的对话和反应 | 与 NPC 互动时 |

### 4.2 技能创作 AI Prompt 设计

**系统提示词（System Prompt）：**
```
你是一位游戏技能设计师。玩家将用自然语言描述他们想要的技能，你需要：
1. 解析技能效果，转化为结构化的游戏数据
2. 从威力、范围、持续性、灵活性、独特性、combo潜力 6个维度评分（1-10）
3. 计算技能点消耗（公式已提供）
4. 生成技能的正式描述文本

注意：
- 保持技能效果与玩家描述一致
- 如果描述过于模糊，合理填充细节
- 如果描述过于强大（如"秒杀一切"），要适当削弱并解释原因
- 返回 JSON 格式
```

## 五、技术架构

### 6.1 技术栈（MVP）

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **前端** | React + Tailwind CSS | 文字冒险界面，响应式布局 |
| **后端** | Node.js + Express | API 服务 |
| **数据库** | PostgreSQL | 玩家数据、技能数据、存档 |
| **AI 接口** | OpenAI GPT-4o / Claude 3.5 | 剧情生成、技能解析、强度评估 |
| **缓存** | Redis | 会话缓存、AI 响应缓存 |
| **部署** | Vercel (前端) + Railway/Render (后端) | 快速上线 |

## 七、风险评估与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| **AI 成本过高** | 高 | 缓存常见技能模板；限制每玩家每日创作次数；使用便宜的模型做初筛 |
| **AI 生成技能不平衡** | 高 | 后端二次校验评分；建立玩家反馈调整机制；人工审核高消耗技能 |
| **玩家创作内容违规** | 中 | AI 内容安全过滤；关键词屏蔽；玩家举报机制 |
| **游戏进程过短** | 中 | AI 动态生成无限剧情；roguelike 机制（每次冒险不同） |
| **留存率低** | 中 | 技能收藏癖驱动；成就系统；社交分享功能 |
