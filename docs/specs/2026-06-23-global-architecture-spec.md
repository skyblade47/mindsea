# AI 肉鸽文字冒险 — 全局架构设计（v1.0）

> 范围：顶层模块划分、接口定义、数据流、扩展点、AI调用策略。
> 不包含：具体数值、具体技能设计、UI细节。

---

## 一、架构总览

### 1.1 核心原则

1. **骨架约束AI，AI填充内容** —— L0规则绝对不可变，L1/L2由AI生成但受验证
2. **前后端分离，后端是权威** —— 所有游戏性计算在后端完成，前端纯展示
3. **AI是生成器，不是引擎** —— AI只负责"写什么"，不负责"怎么算"
4. **状态极简** —— 单次冒险的所有状态可序列化为一个JSON，便于存档/回放

## 二、模块详细定义

### 2.1 冒险引擎 (Adventure Engine)

**职责**：管理单次肉鸽冒险的完整生命周期

### 2.2 创作引擎 (Skill Creator Engine)

**职责**：处理技能创作的全流程，包括AI调用、验证、3选1、重试

### 2.3 战斗引擎 (Combat Engine)

**职责**：处理回合制战斗的完整流程

### 2.4 规则引擎 (Rule Engine)

**职责**：L0绝对骨架的执行与验证

---

## 三、关键数据流

### 3.1 技能创作完整数据流

```
玩家输入 → 预处理 → AI生成(3变体) → 规则验证 → 3选1展示 → 玩家选择 → 扣除资源 → 保存技能
```

### 3.2 冒险推进数据流

```
玩家选择 → 节点解析 → 类型分发 → 执行逻辑 → 结果应用 → 叙事生成 → 状态更新
```

## 四、扩展点设计

| 扩展点 | 当前实现 | 未来扩展 |
|--------|---------|---------|
| **伤害类型** | 6种基础类型 | AI提议新增，后端审核后入库 |
| **效果类型** | 10种基础效果 | 同上 |
| **创作层级** | Lv0-Lv6 | 可增加Lv7+（如"法则编辑"） |
| **敌人类型** | 6种基础模板 | AI生成全新敌人类型 |
| **事件类型** | 7种节点 | 玩家/社区提议新事件模板 |
| **多人模式** | 无 | 预留`party_id`字段，未来支持2-4人协作冒险 |
| **赛季系统** | 无 | 预留`season`字段，未来支持赛季重置+限定内容 |

## 五、前后端边界

### 5.1 前端职责（纯展示+轻量交互）

- 渲染叙事文本（支持打字机效果、关键词高亮）
- 展示技能/装备/属性面板
- 收集玩家输入（技能描述、战斗选择、事件选择）
- 本地缓存冒险状态（断线重连）
- **不计算任何游戏性数值**

### 5.2 后端职责（权威计算）

- 所有数值计算（伤害、消耗、成长、评分）
- AI调用与响应处理
- 规则验证
- 状态持久化
- 反作弊（验证玩家输入的合法性）

## 六、AI 安全与成本控制

### 6.1 输入过滤（防注入）

- 长度限制（防止超长prompt消耗token）
- 敏感词过滤
- Prompt注入检测

### 6.2 输出验证（防幻觉）

- JSON Schema验证
- 数值范围验证
- 语义一致性验证

### 6.3 成本预算（每局冒险）

- 单局总token上限：50000
- 单局总调用次数上限：30
- 降级策略：fallbackModel: 'gpt-4o-mini'

## 七、模块依赖关系

```
冒险引擎 ──→ 创作引擎 ──→ AI服务层
    │           │             │
    │           │             ▼
    │           │        [外部AI API]
    │           ▼
    │        规则引擎 ←─────┐
    │           │           │
    ▼           ▼           ▼
战斗引擎 ←── 数据层 ←── 类型系统
```

**依赖原则**：
- 上层可调用下层，下层不可调用上层
- 同层模块通过接口通信，不直接依赖实现
- AI服务层是唯一的外部依赖，有熔断机制
